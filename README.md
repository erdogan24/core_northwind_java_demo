•	EN — Project Overview

This project—Core Northwind Java—demonstrates a REST-based product management service built with Spring Boot 3.5.x on Java 21. Its purpose is to store products and their categories in a PostgreSQL database and provide reliable CRUD capabilities (listing, searching, creating) through a clean layered architecture (Controller → Service → Repository → Entity/DTO). The domain model maps a many-to-one Product→Category and one-to-many Category→Product relationship using JPA/Hibernate 6, with Spring Data JPA repositories and derived queries (including relationship property navigation such as category.categoryId) to keep queries type-safe. The stack includes HikariCP for connection pooling and springdoc-openapi with Swagger UI for interactive API exploration; GET and POST endpoints have been verified through Swagger. Responses are standardized via Result/DataResult wrappers to ensure consistent API contracts. Overall, the project offers a maintainable, extensible, production-style foundation for a real database-backed product service aligned with modern Java best practices.

•	TR — Proje Özeti

Bu proje, Core Northwind Java adıyla, kurumsal bir ürün-kategori yapısını örnek alarak REST tabanlı bir servis geliştirmeyi amaçlar. Temel görevi; ürünleri ve bağlı oldukları kategorileri PostgreSQL veritabanında tutmak, bu veriler üzerinde listeleme, arama ve ekleme gibi temel CRUD işlemlerini güvenilir ve katmanlı bir mimariyle sunmaktır. Uygulama, Controller → Service → Repository → Entity/DTO akışını izleyen temiz bir yapıdadır: Controller katmanı HTTP isteklerini karşılar ve iş kurallarını barındıran Service’e yönlendirir; Service, veri erişimi için Repository (Spring Data JPA) katmanını kullanır; veri modeli ise Product (çok→bir Category) ve Category (bir→çok Product) ilişkileriyle JPA/Hibernate üzerinde tanımlanır. Teknoloji tarafında proje Java 21 ve Spring Boot 3.5.x ile çalışır; Spring Data JPA / Hibernate 6 ORM katmanını, HikariCP bağlantı havuzunu ve springdoc-openapi ile entegre Swagger UI’yi kullanır. Veritabanı bağlantısı PostgreSQL üzerinden sağlanır; istekler ve yanıtlar, projedeki Result/DataResult sarmalayıcılarıyla tutarlı bir biçimde döndürülür. Geliştirme sürecinde Swagger UI üzerinden GET ve POST uç noktaları başarıyla test edilmiştir; bu sayede hem fonksiyonellik doğrulanmış hem de API’nin keşfi kolaylaştırılmıştır. Sonuç olarak proje, gerçek bir veritabanıyla konuşan, bakımı kolay, genişlemeye açık ve modern Java ekosistemi standartlarına uygun bir ürün yönetimi servisinin somut örneğini sunar.
